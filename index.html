从客户需求演进到金山云 AI 全栈闭环

过去
Chat问答、总结、写作，主要验证模型效果。
现在
Agent规划复杂任务，自主调用 ERP、CRM、数据库等系统。
闭环
Workflow从生成文本走向业务动作闭环，需要工具、权限、数据和运行环境。
平台
AgentKit承接上层智能体应用，把客户数据资产与业务逻辑沉淀在平台。
客户需求演进

来源于企业级客户从简单的“单轮对话 Chat”向“复杂任务规划与自主执行 Agent”转型的战略诉求。企业不再满足于生成文本，而是需要 AI 自主调用内部 ERP、CRM、数据库等系统完成闭环业务。

战略闭环规划

金山云从底层 GPU 智算算力，到中层星流模型平台，再到上层星源 AgentKit，形成 AI 服务全栈闭环，通过智能体层沉淀客户的数据资产与业务逻辑。

企业生产级 Agent 的核心痛点

痛点
开发
连接

能力缺口
高代码门槛：开发者要同时处理 Agent 框架、工具调用、Prompt、状态管理和部署脚本，业务团队难以独立构建。
工具集成碎片化：每个 ERP、CRM、数据库和内部 API 都要写适配代码，鉴权、参数转换、错误处理难复用。
执行环境不隔离：代码、浏览器、文件和外部接口调用缺少统一沙箱，容易产生越权、误操作和安全外溢。
效果不可量化：出错靠人工排查，Trace、Token、工具调用、任务成功率和资源成本缺少统一视图。
业务风险
开发周期长，项目交付依赖少数专家，难以规模复制。
系统打不通，工具越接越乱，后续维护和客户复制成本持续升高。
Agent 被诱导执行危险动作，影响业务系统和客户数据安全。
上线后失控，效果、成本、稳定性和责任边界都难解释。
平台解法
CLI / SDK / 模板工程，屏蔽底层资源与框架差异，让开发者聚焦业务逻辑。

AgentEngine + Sandbox 开箱即用，支持 VM/MicroVM 隔离、资源托管和弹性运行。
OTEL Trace 全链路可视，Agent 评测闭环，日志审计和云原生资源监控统一管理。

2 风控相关2.1 用户数据安全与隐私保护2.2 业务合规性

风控方案：默认隔离、最小授权、全链路可追溯

不涉及数据出境模型推理、记忆、知识库与沙箱环境部署在中国大陆境内数据中心。
不共享租户资源计算、网络、知识库、记忆库和工具凭据按租户隔离。
不开放越权执行Agent 的接口调用、代码执行和日志留存均进入平台治理闭环。
数据与网络 把租户边界落到资源层
  • KCI 容器级隔离，后续演进到 Micro VM 硬件级 + 内核级双重隔离
  • 独立网络命名空间、虚拟网卡、IP 与路由表
  • 默认白名单网络策略，禁止租户实例横向互访
租户数据 把业务数据隔离到组件层
  • 知识库与记忆库按企业/租户分配独立资源空间
  • RAG 文档集、用户记忆、会话数据不可跨租户访问
  • MCP API-Key 与内部路由按账号属性精准隔离
执行边界 把 Agent 能力约束在授权内
  • MCP 双模式鉴权：平台 API-Key + 客户自有 Key
  • Sandbox 限定代码、浏览器、文件和系统调用范围
  • 输入/输出接入内容安全审查，过滤高风险生成内容
审计合规 把过程沉淀成可验收证据
  • Trace 记录模型交互、工具调用、沙箱操作和异常链路
  • 日志留存策略按合规要求配置，可满足不少于 6 个月审计
  • 等保/安全产品许可按最终销售形态补充评估材料
验收抓手
数据不出境说明租户隔离设计Sandbox 边界说明MCP 鉴权策略Trace / 日志留存策略等保与许可评估材料
风控目标：把 Agent 的数据、网络、工具调用和代码执行全部收束在可授权、可隔离、可追溯的边界内Risk Control

法务与安全：外部依赖可控，核心能力自研，客户数据边界清晰

依赖可追溯兼容 MIT / Apache 2.0 等宽松许可生态，建立组件版本与许可清单。
资产可沉淀平台架构、沙箱预热、Agentic 记忆与知识检索形成自研技术资产。
外部依赖 只引入许可边界清晰的生态组件
  • LangChain、LangGraph 等开源生态组件：MIT 协议
  • OpenClaw：Apache 2.0 协议
  • 依赖清单记录组件名称、版本号和许可类型
  • 后续版本新增依赖进入清单维护流程
技术资产 核心平台能力以自研沉淀
  • 平台架构设计、沙箱预热机制为金山自研
  • Agentic 记忆检索、知识库检索具备专利申请价值
  • 当前方案不涉及第三方专利授权依赖
  • 后续可按模块沉淀专利申请材料
数据边界 只处理客户授权上传的业务内容
  • 知识库/记忆库处理用户上传业务信息与会话上下文
  • MCP 代码方式部署会解析处理用户上传代码内容
  • SkillsHub 处理用户主动上传的 Skills 内容
  • 平台处理数据但不拥有客户数据
安全口径：平台会处理客户授权上传的数据与代码内容，但不拥有客户数据；通过依赖清单、技术资产沉淀和数据边界说明控制风险Legal & Security
