模型 API 服务-2026H1 产品力总结

Executive Summary

产品力从“模型中转服务”升级为“企业级调用平台”。 上半年持续补齐模型广场、接入点、API Key、OpenAPI、在线调试、微调、用量统计、计费和额度管理，支撑模型服务规模化、可治理地调用。
经营质量成为产品力的一部分。 5 月初整体毛利率约 6.07%，6 月末提升到 22.66%；自建业务在 6.25-7.1 毛利率从 3.79% 提升至 42.31%，体现出资源调度与计量治理能力的经营效果。
资源调度能力开始沉淀为产品能力。 闲忙时差异化定价、闲时资源调整、水位调度、接入点后端模型切换、Token 额度管控和流式中断计费，共同构成损益提升的产品基础。

H1 产品进展

结合产品动态和月度计划表，上半年进展可以归纳为五类能力。 月度计划表补充了版本内上线需求，尤其是协议兼容、模型生态、内部调度和运营后台能力。

模型生态、接入与路由

重点从“多模型上线”推进到“模型类型、接入方式和路由治理一起补齐”。

多模型/多模态扩容国际版与渠道定价接入点治理微调与模型生态补齐
1月：模型广场上线 qwen3-235b、kimi、qwen3-vl-flash、mgg-7、mgg-6、mos-1、wan2.6-i2v、kling-v2-6、kling-video-o1；新增在线推理接入点，支持 API Key 授权、项目制划分、接入点维度限流和监控；支持 OpenAPI。
2月：国际版 API 服务支持 mc 系列并上线 mco-4；上线 qwen3-30b、glm-5、minimax-m2.5、qwen-flash、qwen-plus、Eleven v3、eleven_flash_v2、wan 2.6 t2i；海外模型采用美元定价；语音模型类别进入模型广场和开通管理。
3月：新增接入点级 Token 额度禁用；接入点监控覆盖国内版/国际版；多集群管理、mgg 模型支持通过接入点 ID 调用；引入 mino v2 pro、minimax m2.7、hunyuan-t1-vision、Tencent HY Vision 1.5 Instruct。
4月：支持接入点后端模型切换；上线 xiaomi mimo-omin、Xiaomi MiMo-V2-TTS、qwen3.5-397b-a17b、kling-v3-omni、qwen3.5-flash、qwen3-embedding-8b、text-embedding-3-small、qwen3-coder-plus、kling-v3；多模态模型支持公网访问；声音克隆接口、Embedding、Coder、视频模型等能力进入模型生态。
5月：支持批量修改和删除接入点；上线 GPT-image-2、可灵 v3/v3 omini、qwen 3.6 plus；适配 Claude CC 新参数、Gemini v1beta API；预置推理点 ID 调用下线；模型微调模块上线，支持任务创建、任务配置、状态管理、详情和监控、SFT、LoRA、标准化数据接入、OpenAPI、日志和事件查询。
6月：上线 speech 系列、glm-5.2、glm-5-turbo、qwen3.6-flash、qwen3.7 max、seedance 2.0-domestic，国际版上线 Eleven v3、eleven_flash_v2、eleven_flash_v2_5、deepseek v4 pro；海外模型支持自定义选择接入渠道并区分定价；已下线模型对应接入点状态置为异常；可灵自定义主体查询、seedance 2.0 计费项、视频/图片生成相关接口继续补齐。

权限与额度：把调用风险前置到产品流程

重点从 API Key 管理扩展到接入点、项目、免费额度、推理限额等多层控制。

API Key 权限接入点额度免费额度预警自动禁用
1月：API Key 权限管理支持自定义接入点权限范围；配额管理支持账号下配额与限制，并可为指定项目分配配额；免费额度使用情况支持云监控提醒；未配置 API Key 时全页面提示。
2月：API Key 数量限制逻辑修改，数量从子用户 20 个提升到主账号 1000 个；KSCC 创建的 API Key 禁止用户通过 OpenAPI 操作；模型广场和开通管理支持语音模型类别。
3月：接入点维度 Token 使用额度禁用；API Key 自定义权限配置支持全选后反选部分模型；接入点禁用流程优化。
4月：开通管理支持调整模型推理限额和报警；开通管理展示开通状态、模型定价信息和模型详情页跳转；接入点设置限额后自动预警。
5月：API Key 权限管控支持管理员配置账号成员仅可编辑自己的 API Key；批量开通模型/API Key 权限管理支持按模型类型筛选；API Key 授权时未开通模型给出提示。
6月：免费额度耗尽预警及通知上线；开通与批量开通增加计费风险提示；免费额度 80%/100% 触发短信、邮件、站内信与页面弹窗，并支持自动禁用。

计量与成本透明：让模型选择和调用成本可见

重点是把模型成本、用量、缓存和计费规则变成可查询、可提示、可解释的产品信息。

用量维度扩展价格字段透明缓存/接入点统计流式中断计量
1月：用量统计增加图片数量、视频时长统计；模型详情页/模型对比页增加名词解释、文档跳转和阶梯计费展示优化。
2月：海外模型采用美元定价，列表价和官方列表价统一。
3月：用量统计支持语音模型信息展示。
4月：/v1/models 接口补充模型上下文长度、最大输出长度、输入/输出类型及价格字段；用量统计支持按接入点、缓存、API Key 等维度查看。
5月：模型广场 hover 模型卡片可查看开通状态、免费额度使用情况、模型信息；开通管理中增加写入缓存不能用免费额度提示。
6月：国内模型流式输出主动中断场景按中断前已成功输出 Token 计量计费。

协议兼容与工具调用能力

重点是减少模型协议差异对调用方的影响，提升工具链兼容度和异常可解释性。

官方参数对齐Cursor / Claude CodeResponse API错误码与 Header 治理
1月：DeepSeek 深度思考参数对齐官方；自建模型支持前缀续写；显式缓存 Claude 相关能力排期推进。
2月：文件查询接口禁用；国际版 mc 系列模型接入后补齐海外调用协议链路。
3月：网关兼容 mog 流式返回 event/data 字段；GLM 深度思考与官方一致；GLM require=null 参数兼容；兼容请求内容未闭合异常；Anthropic 协议 tool_stream 默认为 true；现有错误码统一。
4月：联网搜索兼容智谱官方格式；上下文超限时报错内容透传；GLM-5 思考参数与第三方映射并增加 thinking_adaptive；兼容 Cursor、Claude Code beta 特性；/v1/models 返回内容增强。
5月：支持 Response API；上线在线调用调试；DeepSeek v4 pro reasoning_effort 参数兼容；Gemini 支持 v1beta API；arguments 解析修复；可灵 Callback 接口支持自定义 URL。
6月：DS v4 pro/flash、GLM5/5.1 的 websearch、cache tokens 字段对齐原厂；response header 屏蔽来源 header；mgg 系列原始 error message 重写；可灵查询任务接口支持多租户隔离。

运营与精细化调度能力

重点是把内部流量治理、可观测、限流、自动切流能力产品化，服务损益提升目标。

水位监控闲时调度限流白屏化自动化切流
1月：底层集群资源部署监控、模型多 provider 路由策略查询进入月度重点，开始补齐内部调度可观测基础。
2月：国际版调用、海外定价和 API Key 管控补齐，为后续多版本、多渠道调用治理提供基础。
3月：自建集群水位相关监控与告警、服务监控面板、不可用 provider 自动切换一期、错误码统一进入版本计划。
4月：管理后台能力增强：第三方服务评测能力、服务管理、网关闲时调度逻辑、考虑 TPM 水位的自动调度、主动监测返回内容并中断。
5月：用户限流配置和专属路由限流白屏化、个人账户限流配置、最后修改人与历史记录、路由改造与调度、网关 session 保持能力上线或提测。
6月：专属路由 RPM 限流、长文本负载分发控制、自动化切流一期、服务质量签名验证定时任务等内部调度能力继续推进。

损益趋势：5 到 6 月毛利质量稳步提升

随着 5 到 6 月精细化运营动作逐步落地，毛利质量有了稳步提升。 6.25-7.1 整体收入 820.99 万，成本 634.94 万，毛利 186.06 万，毛利率 22.66%；对比上一周期，收入增长 12.7%，成本下降 3.3%，毛利增加 114.27 万。

模块拆解

自建部分：5.1-5.7 毛利率为 -7.33%，处于亏损状态；6.18-6.24 已回到 3.79%；6.25-7.1 提升到 42.31%，收入 268.11 万、毛利 113.43 万，是最新周期毛利提升的主要来源。

第三方部分：5.1-5.7 毛利率为 9.79%，6.18-6.24 为 11.52%，6.25-7.1 为 13.14%；收入规模从 587.35 万到 552.88 万小幅回落，但毛利从 57.52 万提升到 72.63 万，体现为较稳的毛利率改善。

运营效果：闲时资源能力已经形成可量化收益

闲时资源调整是目前最清晰、可核算的产品能力应用结果。 收益核算逻辑为：按调整后自建调用量乘以“调整后自建占比 - 调整前自建占比”计算净增收益用量，已调整但未经新增调整的夜间自建调用不计入收益。

解读：5.16-5.22 收益达到 67.99 万；5.23-5.28 回落，说明闲时资源能力要继续依赖稳定的夜间调用、准确水位和自动化调度能力。

产品能力基础：推理吞吐提升降低单位资源成本

损益提升不仅来自调度策略，也来自推理吞吐能力提升。 5 到 6 月多模型推理能力持续优化：K2.5 提升约 130%，glm-5 提升约 58%，glm-5.1 提升约 88%，deepseek-v3.2 提升约 100%。

解读：吞吐提升意味着同等资源可承接更多调用，为闲时资源调整、削峰和自建毛利率提升提供基础。

总结、展望与致谢

上半年，模型 API 服务完成了从模型中转到企业级调用平台的关键能力补齐。 模型生态、接入点治理、权限额度、计量透明、协议兼容和精细化调度共同构成了产品力底座。

下半年重点是把这些能力进一步做稳、做自动、做精细。 一方面继续提升模型接入和协议兼容效率，另一方面重点优化大规模用量产生后的用户出账和供应商出账准确性，同时推进多集群、多 provider 的精细化管理和调度，让产品能力持续反映到经营质量上。
