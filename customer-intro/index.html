产品定位

AgentKit 面向企业级 Agent 上线后的基础设施问题，回答两个核心问题：为什么 Agent 需要 Infra，以及 AgentKit 在企业 AI 架构中承担什么角色。

Why Agent Infra

为什么 Agent 需要 Infra？

一个 Agent 做 Demo 不难，真正进入业务使用或对外服务时，难点往往不在“能不能回答”，而在它能否稳定连接系统、执行任务、隔离风险、持续观测和迭代。

What Is AgentKit

AgentKit 是什么？

AgentKit 是面向企业级 Agent 应用的基础设施平台。它不替客户决定 Agent 业务逻辑，而是提供上线和服务化所需的部署、连接、运行和治理能力。

连接统一访问知识库、业务 API、内部系统和第三方工具，减少碎片化适配。
运行托管长链路任务、并发执行、代码运行和 Skill 调用，避免临时机器与脚本运维。
安全通过沙箱隔离、权限控制和边界约束，降低高自主 Agent 的执行风险。
治理回看调用链路、评测效果、定位问题并持续优化，支撑生产级上线。

典型客户使用场景

AgentKit 适合已经明确 Agent 业务目标，但需要生产级运行、工具连接、隔离执行和治理能力的客户。以下场景既可以独立落地，也可以组合形成更复杂的企业 Agent 服务。

Reactive / Tool-Calling

业务查询助手

面向云资源控制台、运维查询、订单状态、产品文档问答等轻量交互场景，Agent 根据用户输入选择知识库或工具完成查询与建议。

Agent Runtime知识库MCP 工具AI 网关
用户问题 → Agent 托管运行 → 知识检索 / MCP 调用 → 模型生成 → 可追踪返回

客户价值：减少人工查文档和查系统的成本，让控制台、客服台、售后台具备自然语言服务能力。

Workflow / Layered Pattern

工作流型业务 Agent

面向招聘、合规初审、售前问答、工单流转、财务核验等有明确 SOP 的场景，将 Agent 决策嵌入结构化业务流程。

客户价值：将可重复流程固化为可调用、可复用、可评估的 Agent 服务，降低人工处理量并提升一致性。

Autonomous / Plan-and-Solve

自主规划与研发辅助 Agent

面向代码分析、漏洞复现、复杂资料调研、自动化验证等高自主性任务，Agent 需要规划、执行、反思并动态修正任务树。

将产品手册、行业规范、制度文档、合同资料、FAQ 和历史工单接入知识库，面向员工、客户或渠道提供可靠问答。

产品能力模块

AgentKit 不是单一开发框架，而是一组面向生产级 Agent 的平台能力组合。客户可以按场景选择模块，也可以采用完整闭环方案。

01 Runtime

Agent 运行时

Agent 的部署、发布、调试和生命周期管理底座。

已上线 / 增强中
核心能力
  • 组件关联、版本管理、回滚与在线调试。
  • 运行详情、资源监控、自动休眠与弹性托管。
  • 支持第三方 Agent 代码部署与容器化发布。
适配场景
  • 已有 Agent 迁移到云上托管。
  • 商业化 Agent 服务的高可用运行。
  • 多 Agent 服务的统一发布与运维。
02 Connection

MCP 广场

适配场景

相比自建 AgentKit 的优势

企业自建 Agent 底座通常需要分别处理工具适配、沙箱、安全、观测和运维问题。AgentKit 将这些生产级能力预先集成，帮助客户更快把 Agent 推向真实业务。

自建 Agent Infra

工具接入每个系统写一套适配代码，鉴权、参数转换、错误处理重复建设。
代码执行自建沙箱周期长，业务机器存在安全隐患。

评测观测出错靠人工排查，Token 消耗、调用链路和效果难以复盘。

客户接入路径

面向正式客户交流，建议从场景识别、架构方案、PoC 验证到生产发布分阶段推进，确保业务价值、资源规格、安全边界和交付节奏一致。

Step 01场景澄清明确 Agent 类型、业务目标、用户规模、工具调用、数据来源和安全要求。
Step 02架构设计选择 Runtime、知识库、记忆库、MCP、Sandbox、网关和评测等模块组合。
Step 03PoC 验证接入样例数据和工具，验证回答质量、任务执行、隔离边界和调用链路。
Step 04生产部署完成资源规格、网络、安全、权限、监控、计费和容量规划配置。
Step 05持续运营基于 Trace / Eval / 网关数据持续优化效果、成本和服务稳定性。
面向客户的完整方案通常包含：Agent 运行架构、工具接入清单、安全隔离方案、资源规格、计量计费方式、上线计划和后续迭代路线。
